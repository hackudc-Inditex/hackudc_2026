Fase 1: Modelo (GPU)

El modelo genera top-40 candidatos por bundle con scores de confianza.

Bundle → YOLO (crops) → CLIP (768d) | Projection Head | FAISS search | candidates.json (40/bundle)

Fase 2: Scoring + Human (CPU)

Re-puntua candidatos con 8 seniales + labels humanas → top 15 final.

candidates.json + human_labels.csv (120 bundles) + SKU / temporal / co-ocurrencia | submission.csv (15/bundle)

Heuristicas multi-senial + active learning

Fase 1: Deteccion de prendas

YOLOv8x DeepFashion2 detecta prendas individuales (13 categorias). Para lo que YOLO no detecta, usamos zone crops.

YOLO detecta

camiseta camisa chaqueta pantalon falda vestido shorts chaleco

YOLO NO detecta → Zone Crops

  • Pies (75-100% inferior) → zapatos (~18% del GT)
  • Cabeza (0-18% superior) → gorros, gafas
Imagen del modelo +---------------------------+ | [ cabeza ] 0-18% | → CLIP → 768d | | | [ YOLO: camiseta ] | → CLIP → 768d | | | [ YOLO: pantalon ] | → CLIP → 768d | | | [ pies ] 75-100% | → CLIP → 768d +---------------------------+ Cada crop → embedding independiente → busqueda dirigida por categoria

Sin zone crops perderiamos todos los zapatos del ground truth.

Fase 1: CLIP + Contrastive Learning

CLIP ViT-L-14

Modelo vision-lenguaje (OpenAI, 400M pares). Genera embeddings de 768 dimensiones que capturan semantica visual.

Embedding de cada crop + embedding de cada producto del catalogo (27K).

Projection Head (MLP)

Transforma embeddings CLIP para que crop y producto correcto esten mas cerca.

768 → Linear → 1536 → GELU → LayerNorm
→ Linear → 768 + residual → L2 norm

~2.4M params, 30 epochs, InfoNCE loss

InfoNCE Loss

Para cada crop, distinguir el producto correcto entre 31 negativos de la misma categoria:

sim(crop, positivo) >> sim(crop, negativo)

L = -log( esim_pos/t / Σ esim_i/t )

Temperatura t = 0.07, AdamW, cosine annealing

FAISS

Busqueda por similaridad coseno en indices segmentados por (categoria YOLO x seccion). Top-40 candidatos por bundle → candidates.json

Fase 2: Scoring multi-senial

Re-puntuamos los candidatos combinando 8 fuentes de evidencia:

1

Human Labels

Anotaciones manuales verificadas (peso maximo)

2

Model Candidates

Scores del modelo (candidates.json) re-ponderados

3

Temporal Neighbors

Bundles de training con timestamp cercano (<90 dias)

4

Co-ocurrencia

Productos que aparecen juntos frecuentemente

5

SKU Exacto / Prefix

Codigo de producto en la URL (variantes de color)

6

Popularidad

Productos frecuentes en la seccion (accesorios boosted)

7

Timestamp

Cercania temporal bundle-producto (misma temporada)

8

Noise Filter

Excluir 53 categorias sin apariciones en GT (perfumes, velas...)

Insights del analisis de datos

Seccion = particion total

Cero solapamiento de productos entre secciones. Restringimos busqueda: 27K → ~9K productos.

Accesorios se reutilizan

Un mismo zapato puede estar en 38 bundles diferentes. Ropa: 1-2 bundles.

53 categorias = ruido

Perfumes, velas, libros... nunca aparecen en fotos. Filtrar 1,649 productos de ruido.

Ventana temporal

Productos con timestamps cercanos (misma temporada). Ropa: ~30 dias; accesorios: ~120 dias.

SKU en URLs

Las URLs contienen codigos SKU. Mismos primeros 8 digitos = variantes de color del mismo articulo.
